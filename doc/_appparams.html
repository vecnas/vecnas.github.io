Id и другие параметры приложения

Идентификатор приложения был введен в 0.20 чтобы поддерживать несколько приложений на странице.

Объявление

Добавьте поле id к спецификации приложения: 

    var todomvcJiant = todomvcJiant || (function (jiant) {
        return {

            id: "todomvc-jiant",

            states: .....
        }
    }

Связывание приложения

Вызовите bindUi. Jiant проверит наличие идентификатора

Использование

Передайте идентификатор приложения как первый параметр в функцию onUiBound: 

    jiant.onUiBound(todomvcJiant.id, function($, app) {
или 

    jiant.onUiBound("todomvc-jiant", function($, app) {

Остальные параметры приложения

Jiant переменная определения приложения может содержать следующие специальные переменные на верхнем уровне:
  • id - объяснено выше
  • ajaxPrefix - см. секцию про аякс, обычно для кроссдомена
  • ajaxSuffix - см. аякс
  • stateExternalBase - см. состояния
  • crossDomain - см. кроссдомен
  • dirtyList - массив идентификаторов виджетов, для которых игнорируются ошибки связывания. Не рекомендуется к использованию, но может использоваться чтобы спрятать вашу лень и склонность к грязному коду
  • appPrefix - определяет строковый префикс, добавляемый к идентификатору или классу при поиске виджетов и их компонентов в HTML, замена для первого параметра в методе bindUi
Например: 

    var myApp = {
        id: "myApp",
        ajaxPrefix: "http://test.com:8080/ajax/",
        ajaxSuffix: ".do",
        crossDomain: true,
        appPrefix: "_",
        dirtyList: ["dirtyView", "_dirtyTemplate"]
    }

Логгирование

Когда идентификатор приложения отсутствует - jiant по-прежнему работает, создается идентификатор по умолчанию и печатается ошибка в консоль: 

!!! Application id not specified. Not recommended since 0.20.
    Use 'id' property of application root to specify application id
