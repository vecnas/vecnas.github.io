Типы элементов

Jiant предлагает несколько типов для определения элементов виджетов и шаблонов. Большинство носит декоративный характер и служит исключительно целям документирования и читабельности. Но несколько добавляют дополнительный функционал. Полный список всех типов:
  • collection, - коллекция элементов, декоративный
  • container, - контейнер элементов, декоративный
  • containerPaged, - не следует использовать, будет удален
  • ctl, - некий контрол для взаимодействия, декоративный
  • fn, - некая функция с одним параметром. эксперимент, декоративный, будет удален
  • form, - см. далее
  • grid, - таблица, декоративный
  • image, - декоративный
  • input, - элемент пользовательского ввода, декоративный
  • inputInt, - см. далее
  • inputFloat, - см. далее
  • label, - некий текст на экране, декоративный
  • lookup, - см. далее
  • meta, - мета флаг, не элемент интерфейса
  • on, - регистрация коллбэка, декоративный, будет удален
  • pager, - см. далее
  • slider, - декоративный
  • stub, - см. далее
  • tabs, - см. далее

jiant.form

Проверяет что ссылается на HTML form элемент. Когда объявлен как элемент виджета - проверяет на старте приложения, когда объявлен как элемент шаблона - проверяет во время генерации шаблона. В случае ошибки выводит в консоль (и алертом в режиме разработки): 

    myView.myForm form element assigned to non-form: div
если ссылается не на форму Также элемент, определенный как jiant.form, получает метод 

    submitForm(url, callback)
Этот метод использует кроссдоменные настройки и установленный ajaxPrefix (см. раздел Ajax) Если первый параметр неопределен (или равен null, false, 0) - для отправки формы используется ее атрибут action When first parameter is undefined (or null or false or 0) - form attribute "action" used for form submit. Internally this method calls and returns result of 

    $.post(url, form.serialize(), callback);
Example: 

    var myApp = {

        views: {
            main: {
                myForm: jiant.form
            }
        },

        templates: {
            tm: {
                tmForm: jiant.form
            }
        }
    }


    jiant.bindUi("_", myApp);


    myApp.views.main.myForm.submitForm(null, function() {
        alert("form submitted");
    });
    var tmResult = myApp.templates.tm.parseTemplate({});
    // append, manipulate tmResult
    tmResult.tmForm.submitForm("some_specific_url.php");

jiant.inputInt

добавляет ограничение на поле ввода - только цифры, добавляет возможность изменять текущее значение стрелками

jiant.inputFloat

то же что inputInt, плюс пользователь может нажимать точку, чтобы вводить дробные значения

jiant.tabs

вызывает функцию .tabs() на элемент, если она доступна (например, jqueryui есть на странице) Добавляет функцию refreshTabs() к элементу, синоним для вызова uiElem.tabs("refresh");

jiant.stub

выводит имя функции откуда было вызвано алертом. Эксперимент, возможно будет удалено

jiant.lookup

Игнорирует стартовую проверку существования элемента. Является функцией, в отличие от других элементов! Осуществляет поиск элемента во время вызова. Полезно для динамического поиска элементов виджета. Во время связывания пишет в консоль (если включен режим разработки): 

    loookup element, no checks/bindings: elemName

jiant.pager

Эксперимент по построению сложного поведения вокруг одного контрола. Прижилось. Используется в частности в Jiant.XL.

Выстраивает внутри заданного элемента bootstrap-совместимую структуру навигации по страницам, добавляя элементу следующие методы: 

    onValueChange(callback);
    updatePager(page);
объект страницы page является Spring-совместимым и должен иметь следующую структуру: 

    page: {
        number : 2,
        totalPages: 10
    }
Пример: 

    var myApp = {

        ajax: {
            getData: function(pageNum, callback);
        },

        views: {
            list: {
                container: jiant.container,
                pager: jiant.pager
            }
        }

    }

    jiant.bindUi("_", myApp);

    myApp.views.list.pager.onValueChange(function(event, pageNum) { // note event parameter here is first!
        myApp.ajax.getData(pageNum, function(page) {
        myApp.views.list.pager.updatePager(page);
        // do actual list data update
    });
Как показывает этот пример - все возможности Jiant естественным образом дополняют и усиливают друг друга
