Jiant функции

Список функций jiant: 



    bind(obj1, obj2)
копия $.extend(obj1, obj2). Анахронизм ранних времен, будет удалено 



    bindUi(prefix, root, devMode, viewsUrl, injectId)
выполняет связывание приложения с реализацией. См. соответствующий раздел для деталей 



    formatDate(date)
    formatDateUsa(date)
    formatTime(date)
    formatTimeSeconds(date)
    formatMoney(amount, grpDelim)
разнообразные функции форматирования времени, даты и денег 



    getAwaitingDepends()
возвращает список нереализованных зависимостей, ожидаемых каким-либо кодом приложения 



    goRoot([appId])
переводит приложение в ближайшее корневое состояние. Если appId не передан - переводятся все приложения на странице 



    goState(params, preserveOmitted)
ничего не делает, анахронизм ранних времен. Будет удален 



    onUiBound([app1, app2], [[dep11], [dep21, dep22]], function($, app1, app2) {})
Регистрирует коллбэк на момент связывания приложения. Коллбэк срабатывает когда приложение полностью загружено. Вызывать функцию можно в любой момент - до и после связывания. См. соответствующий раздел для деталей 



    onAppInit(appId, cb)
Регистрирует коллбэк на момент инициализации приложения, после связывания, но до вызова любых onUiBound. Должен вызываться до вызова bindUi, т.е. до связывания приложения. 



    refreshState([app|appId])
Обновляет текущее состояние приложения, см. раздел Состояния для деталей 



    handleErrorFn(errorDetails)
обработчик ошибок серверных запросов для всех серверных вызовов. Приложения могут заменить эту функцию своей реализацией. По умолчанию просто печатает ошибку в консоль. 



    setUiFactory(factory)
способ реализации виджетов и шаблонов можно изменить. Отвечает за это UiFactory. По умолчанию виджеты и шаблоны находятся в HTML коде по идентификатору, а их элементы по классу. jiant_stub_uifactory.js реализует способ автогенерации элементов, если их нет в HTML, достаточно просто подключить этот скрипт. При желании можно установить свою фабрику. Нужно реализовать 3 метода, см. исходники. 



    lfill(val)
Дополняет значение символом "0" слева до длины 2. "2" станет "02". 



    logInfo(message, message2, ...)
выводит сообщение или объект переданный параметром в консоль, если включен режим разработки. При выключенном режиме ничего не делает. См. Режим разработки для деталей. Принимает любое количество аргументов и все их печатает 



    logError(error, arg2, arg3, ....)
печатает все аргументы в консоль с уровнем error. Работает всегда (если есть консоль), но может испугать пользователя красным цветом в консольной печати. Принимает любое количество аргументов и все печатает. 



    info(message, message2, ...)
    error(arg, arg2, ...)
то же что logInfo, logError, но печатает все параметры в одну строку, поэтому сложные объекты будут напечатаны как [Object] 



    parseTemplate(template, data)
Может использоваться отдельно от движка шаблонов, чтобы разобрать отдельные шаблонные строки. Использование jiant.templates гораздо более удобно. Возвращает jQuery обертку результата разбора. 



    parseTemplate2Text(template, data)
аналогично предыдущей функции, но возвращает текстовую строку 



    randomIntBetween(from, to)
возвращает случайное целое число между from и to 



    version()
возвращает версию библиотеки jiant. Используется чтобы загружать самую свежую, если несколько представлено на странице 



    visualize()
Отрисовывает текущее приложение как граф. Скорее для веселья, но может быть полезно чтобы взглянуть на какие-либо тонкости. Вызовите из консоли: jiant.visualize(). После созерцания красоты скорее всего понадобится обновить страницу в браузере. Требует несколько дополнительных скриптов, расположенных на гитхабе: arbor.js, arbor-tween.js, graph.js
