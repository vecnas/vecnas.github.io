Режим разработки

Выставление режима разработки посредством флага DEV_MODE включает дополнительную функциональность и добавляет отладочные сообщения в консоль для облегчения понимания что происходит или почему что-то не работает

Начало

3 способа включить режим разработки:
  1. установить флаг jiant: 
    
                    jiant.DEV_MODE = true
  2. передать true как третий параметр вызова bindUi: 
    
                    jiant.bindUi("_", myAppRoot, true);
  3. Добавить в урл (в любое место) строку "jiant.dev_mode" 
    
                    http://localhost?jiant.dev_mode
                    http://localhost/?lot=of&other=query¶ms=here&jiant.dev_mode
Last one useful for production environment, for example. Second is most convenient.

Логгирование

jiant.logInfo работает только когда DEV_MODE включен. Включается печать всего процесса связывания bindUi() - состояния, события, виджеты, шаблоны, аякс функции и тд. Если режим разработки включен, но сообщения о процессе связывания отсутствуют - возможно флаг был выставлен после вызова bindUi. Примеры сообщений: 

    binding UI for view: dailyBonus jiant.js:147
    bound UI for: container jiant.js:147
    bound UI for: ctls
    ajax bindings:

    binding ajax for function: getUserHomeInfo
    binding ajax for function: getUserStatistics
    binding ajax for function: i18n

Алерт

В режиме разработки если были какие-то ошибки связывания - выводится финальный алерт с полным списком ошибок, чтобы разработчик их точно не упустил.

Информация по щелчку

В режиме разработки каждый UI элемент, связанный с помощью jiant, получает специальный обработчик щелчка мышью:
  • alt+shift+left mouse click - показывает объявление элемента и печатает его в консоль
  • ctrl+alt+shift+left mouse click - дополнительно показывает обработчики событий на элементе и также печатает их в консоль
Достаточно удобно когда что-то не срабатывает, чтобы понять - нет обработчика щелчка на элементе, или нужно искать другую ошибку.
