Начало

Jiant обеспечивает встроенную шину событий. Чтобы использовать ее - добавьте секцию events в определение приложения и опишите ваши события уровня приложения: 

    var myApp = {

        events: {
            NewDataArrived: {
                fire: function(data) {},
                on: function(cb) {}
            },
            NewPhoneCall: {
                fire: function(phoneNumber, operator) {},
                on: function(cb) {}
            }
        }

    }

Подписка на события


    myApp.events.NewDataArrived.on(function(data) {
        //обрабатываем данные
    });

    myApp.events.NewPhoneCall.on(function(phoneNumber, operator) {
        //что-нибудь делаем
    });

Генерация событий


    myApp.events.NewDataArrived.fire(myData);
    myApp.events.NewPhoneCall.fire("1234567890", "Johnny");
Подписчики начнут получать события только после своей регистрации на события. Любое количество подписчиков возможно и кто угодно может генерировать события.

Для совсем ленивых

методы fire и on определены только для целей автозавершения, документации и удобства. Кратчайшая работающая форма шины событий следующая: 

    var myApp = {

        events: {
            NewDataArrived: {},
            NewPhoneCall: {}
        }

    }
Однако не рекомендуется так писать, потому что неясно какие именно данные поступают когда событие случится.

Логгирование

Только когда включен DEV_MODE.

Для каждого определения события в консоль печатается строка (во время стартового связывания): 

    binding event: eventName
Для каждого подписчика в консоль печатается (на каждый on, в любой момент жизни приложения): 

    assigning event handler to eventName

Детали реализации

Каждый объект события также содержит поле listenersCount, увеличиваемое при каждом новом подписчике. Объект самого события сейчас не передается в метод on, поэтому параметры коллбэка on в точности совпадают с параметрами метода fire
