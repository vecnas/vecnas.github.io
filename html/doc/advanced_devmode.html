Режим разработки

Выставление режима разработки посредством флага DEV_MODE включает дополнительную функциональность и добавляет отладочные сообщения в консоль для облегчения понимания что происходит или почему что-то не работает

Начало

Следующим образом можно включить режим разработки
  1. установить флаг jiant: 
    
                    jiant.DEV_MODE = true
  2. Добавить в урл (в любое место) строку "jiant.dev_mode" 
    
                    http://localhost?jiant.dev_mode
                    http://localhost/?lot=of&other=query¶ms=here&jiant.dev_mode

Логгирование

jiant.logInfo работает только когда DEV_MODE включен. Следующее логгирование выводится когда включен режим разработки
  • jiant.pick - выводит сообщения только при включенном режиме разработки
  • сообщение о связывании lookup элемента виджета
  • сообщение о перекрытии именем элемента функции, определенной jQuery, например width и тому подобные имена
  • уведомление об отсутствующем элементе, объявленном с флагом jiant.optional
  • предупреждение об использовании старого формата моделей, без секции jRepo
  • сообщение о начале загрузки внешней библиотеки функцией declare или loadLibs
  • сообщение об окончании загрузки внешней библиотеки функцией declare или loadLibs со временем загрузки
  • сообщение о начале загрузки CSS стиля функцией loadCss
  • сообщение о начале загрузки i18n
  • сообщение о начале загрузки модуля с информацией об инициаторе загрузки и урле модуля
  • сообщение о регистрации модуля
  • сообщение о вызове event.off() без хэндлера, на случай если это было сделано ошибочно

Алерт

В режиме разработки следующие сообщения выводятся через javascript alert
  • Сообщение о том что элемент объявленный как jiant.form привязан не к элементу DOM form
  • Суммарное сообщение об ошибках связывания, если произошли какие-либо ошибки
  • Сообщение об отсутствии плагина jquery.hashchange, если используются состояния
  • Результат выполнения функции jiant.check, в случае если она должна сообщить об ошибке
