Настройки приложения

Jiant переменная определения приложения может содержать следующие специальные переменные на верхнем уровне:
id идентификатор приложения, см. далее
appPrefix определяет строковый префикс, добавляемый к идентификатору или классу при поиске виджетов и их компонентов в HTML, замена для первого параметра в методе bindUi
ajaxPrefix префикс для аякс вызовов, обычно для кроссдомена, подробно описано в секции про ajax
ajaxSuffix суффикс для аякс вызовов, подробно описано в секции про ajax
crossDomain включает кроссдоменные настройки для аяксовых вызовов, также см. секцию про ajax
modulesPrefix префикс для урлов модулей, подробно описано в секции про модули
modulesSuffix суффикс для урлов модулей, подробно описано в секции про модули
modulesTimeout время, которое дается на загрузку одного модуля, по умолчанию 15 секунд
stateExternalBase префикс для состояний, подробно описано в секции про состояния
dirtyList массив идентификаторов виджетов, для которых игнорируются ошибки связывания. Не рекомендуется к использованию, но может использоваться чтобы спрятать вашу лень и склонность к грязному коду
dateFormat формат даты, используется для вызовов datepicker, на элементах типа jiant.inputDate
formatGroupsDelim разделитель десятичных групп при форматировании элементов jiant.numLabel
viewsUrl аналог параметра при вызове bindUi, для загрузки html реализации с удаленного сервера и встраивания в страницу, используется при вызове jiant.app
injectId id элемента для добавления html содержимого с удаленного сервера, аналог параметра функции bindUi, используется при вызове jiant.app
handleErrorFn обработчик ajax ошибок для данного приложения, см. работу с сервером
Например 

    var myApp = {
        id: "myApp",
        ajaxPrefix: "http://test.com:8080/ajax/",
        ajaxSuffix: ".do",
        crossDomain: true,
        appPrefix: "_",
        dirtyList: ["dirtyView", "_dirtyTemplate"]
    }

Идентификатор приложения

Каждое приложение должно иметь идентификатор - строковое значение. Добавьте поле id к спецификации приложения: 

    var todomvcJiant = {

        id: "todomvc-jiant",

        ....
    }
Вызовите bindUi. Jiant проверит наличие идентификатора и сообщит в консоль в случае отсутствия.

Передайте идентификатор приложения как первый параметр в функцию onUiBound: 

    jiant.onUiBound(todomvcJiant.id, function($, app) {
или 
jiant.onUiBound("todomvc-jiant", function($, app) {

Логгирование

Когда идентификатор приложения отсутствует - jiant по-прежнему работает, создается идентификатор по умолчанию и печатается ошибка в консоль: 
!!! Application id not specified. Not recommended since 0.20.
    Use 'id' property of application root to specify application id
