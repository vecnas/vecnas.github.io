Настройки приложения

Jiant переменная определения приложения может содержать следующие специальные переменные на верхнем уровне:
id Уникальный идентификатор приложения, обязательный параметр, см. ниже
appPrefix определяет строковый префикс, добавляемый к идентификатору или классу при поиске виджетов и их компонентов в HTML, замена для первого параметра в методе bindUi. См. также Детали реализации
ajaxPrefix префикс для аякс вызовов, обычно для кроссдомена, подробно описано в секции про ajax Конфигурация приложения
ajaxSuffix суффикс для аякс вызовов, подробно описано в секции про ajax Конфигурация приложения
crossDomain включает кроссдоменные настройки для аяксовых вызовов, также см. секцию про ajax Конфигурация приложения
modulesPrefix префикс для урлов модулей, подробно описано в секции про модули Конфигурация секции
modulesSuffix суффикс для урлов модулей, подробно описано в секции про модули Конфигурация секции
modulesTimeout время, которое дается на загрузку одного модуля, по умолчанию 15 секунд
stateExternalBase префикс для состояний, подробно описано в секции Внешний префикс для состояний
dirtyList массив идентификаторов виджетов, для которых игнорируются ошибки связывания. Не рекомендуется к использованию
dateFormat формат даты, используется для вызовов datepicker, на элементах типа jiant.inputDate
formatGroupsDelim разделитель десятичных групп при форматировании элементов jiant.numLabel
viewsUrl аналог параметра при вызове bindUi, для загрузки html реализации с удаленного сервера и встраивания в страницу, используется при вызове jiant.app
injectId id элемента для добавления html содержимого с удаленного сервера, аналог параметра функции bindUi, используется при вызове jiant.app
handleErrorFn обработчик ajax ошибок для данного приложения, см. работу с сервером
Например 

    var myApp = {
        id: "myApp",
        ajaxPrefix: "http://test.com:8080/ajax/",
        ajaxSuffix: ".do",
        crossDomain: true,
        appPrefix: "_",
        dirtyList: ["dirtyView", "_dirtyTemplate"]
    }

Идентификатор приложения

Каждое приложение должно иметь идентификатор - строковое значение. Добавьте поле id к спецификации приложения: 

    var todomvcJiant = {

        id: "todomvc-jiant",

        ....
    }
Вызовите jiant.app. Jiant проверит наличие идентификатора и сообщит в консоль в случае отсутствия.

Передайте перменную определения приложения как первый параметр в функцию onUiBound: 

    jiant.onApp(todomvcJiant, function($, app) {
или идентификатор, если нет ссылки на переменную 

    jiant.onApp("todomvc-jiant", function($, app) {
