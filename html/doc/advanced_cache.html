Кэширование

Начиная с версии 4.02 Jiant предоставляет возможность "супер" кэширования модулей в local storage.
В этом случае их код сохраняется в local storage и далее берется оттуда, без обращения на сервер.

Сам jiant использует этот механизм для хранения собственных модулей.

Использование

Убедитесь что версия jiant >= 4.02.

В переменной приложения до его связывания определите поле cacheInStorage равным например текущей версии вашего приложения.

Если у приложения есть поле с версией, самый простой способ сделать так: 

    const myapp = {
      ...
    };
    myapp.cacheInStorage = myapp.version;

Если версии в явном виде нет, пропишите это значение прямо в переменной: 

    const myapp = {
      id : "myapp",
      cacheInStorage : "1.0",
      ...
    };
Все, теперь кэш работает.

Обновление модулей

Модули в кэше обновляются автоматически при смене версии. Ключ к кэшированному значению состоит из трех частей - идентификатор приложения, имя модуля и версия, прописанная в поле cacheInStorage. 

  moduleName + "_" + appRoot["id"] + "_" + appRoot["cacheInStorage"];
Таким образом модули с одинаковым именем для разных приложений не переписывают друг друга. Обновить код модуля той же версии приложения - нельзя, кроме как очисткой по ключу в localStorage. Изменение кода любого модуля требует изменения версии приложения, что по моему мнению есть правильное требование.

Отключение кэша

Если кэш нужно отключить (в режиме разработки, когда модули постоянно меняются, или по какой-либо иной причине) - приравняйте поле cacheInStorage нулю или false или просто удалите ее из переменной приложения.

Очистка кэша

Для очистки кэша достаточно полностью очистить localStorage.
