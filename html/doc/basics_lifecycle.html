Жизненный цикл приложения

Приложение jiant имеет несколько этапов жизни, описанных далее, понимание порядка их выполнения позволит выстроить правильные зависимости javascript кода.

Уровень приложения
  1. Абстрактное объявление - на первой стадии, до вызова jiant.app приложение существует в виде абстрактной переменной javascript, в это время любые его изменения повлияют на результат дальнейшего связывания. В частности, можно добавить новые модули в приложение 
    app.modules.push("i18nLoader");
    Наиболее подходящий способ это делать, через вызов jiant.preApp 
    jiant.preApp(appOrAppId, function($, app) {
    app.modules.push("i18nLoader");
    app.modules.push("someModule");
});
    jiant.preApp проверяет что приложение еще не загружено и не в состоянии загрузки
  2. jiant.app(app), загрузка модулей - после вызова jiant.app вначале происходит загрузка модулей. На этой стадии уже нельзя добавлять новые модули в приложение. Модификации остальных частей приложения по-прежнему возможны. Начиная с этого момента вызов jiant.preApp приведет к ошибке. Вызываются все объявленные через jiant.module() модули, в порядке объявленных зависимостей и порядка загрузки. На этой стадии отдельные модули могут добавлять в приложение требуемые для них виджеты, модели и так далее. 
    jiant.module("someModule", function($, app) {
    app.models.someData = {.....};
    app.views.someView = {.....};
});
  3. jiant.app(app), связывание - после загрузки всех модулей происходит собственно связывание приложения. Начиная с этого момента изменения абстрактной спецификации приложения не оказывает влияния на результат связывания. На этой стадии реализуются все объявленные элементы приложения и ищется HTML реализация виджетов и шаблонов после чего выполняются все зарегистрированные коллбэки jiant.onApp. Расширим предыдущий пример 
    jiant.module("someModule", function($, app) {
    var dataModel = app.models.someData = {.....};
    var view = app.views.someView = {.....};
    jiant.onApp(app, function() {
        view.propagate(dataModel);
        dataModel.....
    });
});
  4. jiant.app(app), связывание, зависимости - коллбэки jiant.onApp выполняются в порядке объявленных зависимостей от логик. Данный код выполнится когда будет доступна и реализована логика baseLogic 
    jiant.onApp(app, ["baseLogic"], function($, app) {
        view.propagate(dataModel);
        dataModel.....
    });
});
