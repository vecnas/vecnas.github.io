Библиотечные модули

С помощью модульной загрузки jiant возможно загружать также внешние javascript библиотеки, файлы CSS стилей или html ресурсы. При этом гарантируется заданный порядок исполнения скриптов или применения стилей.

Чтобы использовать эту возможность, модуль объявляется следующим образом 


  jiant.module("extLib", [], {css: "css/theme12.css"});

  jiant.module("libs3d", {js: [
        "https://cdnjs.cloudflare.com/ajax/libs/three.js/98/three.js",
        "/resources/js/3d/Detector.js",
        "/resources/js/3d/TrackballControls.js"
        ]});
Отличие от собственных модулей только в том, что последним параметром передается не функция описывающая модуль, а объект, описывающий библиотеку. Стили и скрипты добавляются и выполняются только один раз на веб страницу, html код добавляется на каждый вызов.

Объявление библиотечного модуля

Следующее объявление включает все возможности библиотечного модуля, каждый из параметров может быть строкой или массивом строк. 


  jiant.module("extLib", [], {
      js: "//myserver.com/resources/js/libs/jquery_ui.js",
      css: "css/theme12.css",
      html: "html/12.html"
  });
Ресурсы загружаются при использовании модуля в рамках приложения или через loadModule и вставляются в текст документа в заданном порядке, вначале html, затем css, затем javascript код.

Использование

Используются библиотечные модули точно так же как обычные, 

  app = {
    ......
    modules: ["moduleApp0", "extLib"]
  }
На них также можно ставить взаимные зависимости и порядок загрузки.

Пример конфигурации


  modules: [
          {"external0" : {
                  order: 30,
                  name: "m1",
                  path: "modules/m1"
              }
          }
