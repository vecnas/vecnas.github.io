Проектирование

Здесь изложена последовательность действий при проектировании и реализации проекта, выработанная в процессе использования jiant

Название и структура

  1. Придумываем название проекту, это будет идентификатором, например demoProject
  2. В папке js создаем папку с названием проекта, в ней создаем папку app, в которой будет размещено приложение и рядом js файл с названием проекта, и закинем первый файл приложения: 
    
            js
            |- demoProject
                    |- demoProject.js
                    |- app
                        |- main.js
  3. файл demoProject.js содержит jiant-определение проекта, 
    
            var demoProject = {
                id: "mycompany.demoProject",
                views: {},
                templates: {}
            }
  4. main.js пока что содержит только пустой коллбэк: 
    
            jiant.onUiBound(demoProject, [], function($, app) {
                alert("Application started");
            });

Хост страница

Создаем страницу, где будет размещено приложение - html или jsp, в конце добавляем нужные скрипты: 

    <script src='js/jquery.js'></script>
    <script src='js/jiant.js'></script>
    <script src='js/jiant_xl.js'></script>
    <script src='js/demoProject/demoProject.js'></script>
    <script src='js/demoProject/app/main.js'></script>
    <script>
        $(function() {
            jiant.bindUi("_", demoProject, true);
        })
    </script>
Сюда же будем добавлять скрипты приложения по мере их появления и html реализации виджетов и шаблонов.

Начальный анализ

Решаем, будет ли приложение поддерживать состояния и хэш-навигацию, будет ли у него меню. Если да, добавляем jiant.xl вызовы в main: 

    jiant.xl.statefulApp(demoProject, "View", app.states.main)();
    jiant.xl.nav(demoProject, app.views.nav, "Ctl", "active")();
Накидываем состояния на глазок в файл с определением 

    states: {
        "": {},
        main: {
            go: function(id) {},
            start: function(cb) {},
            end: function(cb) {}
        },
        profile: {
            go: function() {},
            start: function(cb) {},
            end: function(cb) {}
        }
добавляем виджет с меню и закидываем в него нужные элементы 

    views: {
        nav: {
            mainCtl: jiant.ctl,
            profileCtl: jiant.ctl
        }
    }
накидываем виджеты экранов. 

    views: {
        mainView: {},
        profileView: {}
    }
Теперь у нас есть полноценное приложение с навигацией и меню. Нужно еще реализовать виджеты в html, делаем это 

    <ul id="_nav">
        <li class="_mainCtl">Main</li>
        <li class="_profileCtl">Profile</li>
    </ul>
    <div id="_mainView">Main view</div>
    <div id="_profileView">Profile view</div>

Кроссдоменные виджеты

Когда приложение может использоваться кроссдоменно, следующая организация кода выглядит наиболее удобно. В случае java имеем хостинг jsp и отдельную views.jsp, которая содержит все виджеты и шаблоны. Хостинг jsp импортирует views.jsp, и добавляем метод на сервере, который отдельно может вернуть только views.jsp для кроссдоменной загрузки.

Дальнейшее проектирование

  • Все серверные вызовы по мере появления добавляем в demoProject.js, секцию ajax
  • По мере разработки - добавляем нужные виджеты и шаблоны
  • Если где-то показываем постраничные списки - воспользуемся 
    
            jiant.xl.pagedContent
  • Если какие-то данные важны для разных частей приложения - добавляем models и заносим их туда
  • Если есть какая-то общая логика, не зависящая от специфики приложения - например создание и показ диалогов - лучше вынести ее как logic и впоследствии если что-то поменяется - это сэкономит очень много сил и времени

Разделяй и властвуй

Ни в коем случае не пихайте весь код в один файл приложения. Создавайте много мелких, каждый в идеале до страницы размером и охватывает только небольшую часть.

Минификация

При данной структуре - просто натравите минификатор на папку app
